## Commands

```bash
npm run dev          # Start dev server with HMR (node ace serve --hmr)
npm run build        # Production build
npm test             # Run all test suites (Japa)
npm run typecheck    # Type-check both backend and Inertia TS
npm run lint         # ESLint
npm run format       # Prettier

# Run a specific test suite
node ace test unit
node ace test functional

# Database
node ace migration:run    # Run pending migrations (also regenerates database/schema.ts)
node ace migration:fresh  # Drop all tables and re-run migrations

# Docker (uses PostgreSQL + Redis)
docker-compose up
```

## Architecture

This is an **AdonisJS 7** app with a **React + Inertia** frontend. The backend serves two distinct clients simultaneously: a browser app via Inertia/session auth, and a mobile app via a JSON API with Bearer token auth.

### Dual auth surfaces

- **Web (session)**: `start/kernel.ts` named middleware `guest` and `auth` guard the Inertia routes (`/login`, `/signup`, `/logout`). Uses `web` auth guard.
- **API (Bearer token)**: `middleware.apiAuth()` uses the `api` guard. All mobile routes live under `/api/v1` and `/api/v1/auth`. Tokens expire in 30 days, minted by `UsersController`.

### JSON API response contract

`providers/api_provider.ts` adds `ctx.serialize(data)` to every `HttpContext`. Routes that call `ctx.serialize(...)` return `{ data: ... }`; routes that call `response.ok(...)` / `response.created(...)` directly return unwrapped JSON. See `ROUTES.md` for the full table of which routes wrap and which don't.

### Data flow for write operations

Controllers delegate business logic to services (`app/services/*`). After mutating a game result or stat, controllers emit `GameResultUpdated` (from `app/events/game_updated.ts`). The `UpdateStandings` listener (`app/listeners/update_standings.ts`) recalculates standings for both teams and broadcasts a `game_updated` message over Transmit (SSE) to subscribers of `games/{id}`.

### Model layer

Lucid models in `app/models/*` extend auto-generated schema classes from `database/schema.ts`. **Do not edit `database/schema.ts` by hand** — it is regenerated by `node ace migration:run`. Snake_case DB columns are camelCased automatically (e.g., `full_name` → `fullName`).

### Transformer layer

All API serialization goes through `app/transformers/*`. Transformers are the single source of truth for what fields are exposed in responses. The `app/types/` directory holds domain-level TypeScript types used across services and transformers.

### Routing

Routes are defined in `start/routes.ts`. Controllers are referenced via the generated `controllers` registry (`#generated/controllers`) from Tuyau — never import controller classes directly. Named middleware is registered in `start/kernel.ts`.

The `leagueOwner` middleware resolves `leagueId` from (in order): URL param `params.leagueId`, request body/query `leagueId`, or the parent resource's `leagueId` (for game/stat updates).

### Validation

Validators live in `app/validators/*`. VineJS is used throughout. Global helpers in `app/validators/common.ts`:
- `resourceId(table)` — positive integer that must exist in that table
- `optionalResourceId(table)` — same but optional
- `optionalImage()` — file validation (2 MB, jpg/jpeg/png/webp)

Date inputs are globally converted to Luxon `DateTime` objects in `start/validator.ts`.

### Database

- **Local/test**: SQLite at `tmp/db.sqlite3` (`DB_CONNECTION=sqlite`, default)
- **Docker**: PostgreSQL 16 + Redis 7 (`docker-compose.yml`)
- Test suite (Japa) starts an HTTP server automatically for functional/browser suites; the `.env.test` file configures the test environment.

### TS path aliases

Always use `#`-prefixed imports defined in `package.json` (`#models/*`, `#validators/*`, `#middleware/*`, `#transformers/*`, `#services/*`, `#generated/*`, etc.). Never use relative paths across module boundaries.

### Inertia frontend

Frontend entry: `inertia/app.tsx`. Shared data (auth user, flash messages) is injected by `InertiaMiddleware.share()` and consumed from `inertia/layouts/default.tsx`. Use named routes with Inertia helpers rather than hardcoded URLs.

### Do not edit

- `database/schema.ts` — generated from migrations
- `.adonisjs/*` — build outputs from Tuyau/AdonisJS code generation
